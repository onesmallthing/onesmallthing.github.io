Archived Blog Highlights

  • Devops for SQL Databases. Database DevOps extends software devops principles to databases, enabling automated and repeatable database updates. Two approaches exist: Migration-Based, which uses incremental scripts to apply changes, and State-Based, which defines the desired state and lets frameworks apply the changes.
  • SQL Server Kerberos and SPN Quick Reference This is a comprehensive guide on troubleshooting SQL Kerberos issues. It provides an understanding of Kerberos and Service Principal Names (SPNs), their usage, and the need for SPNs in SQL Server. It also explains how to determine if a SQL connection is using Kerberos or NTLM and discusses the concept of ‘double hop’ and covers tools for configuration and troubleshooting, like SETSPN, aiming to be a central resource on the topic.
  • Guidelines for Setting SQL Permissions on Mount Point Folders The article provides guidelines for setting SQL permissions on mount point folders. It explains that permissions on mount-point root folders are not inherited like regular folders, leading to potential confusion. The recommended approach is to create a subfolder under the mount-point root and set permissions on that, or if necessary, set volume permissions directly.
  • Understanding User Account Control (UAC), A Brief Summary The article provides a brief summary of User Account Control (UAC), explaining the difference between permissions (which apply to objects) and rights (which apply to user account actions). It discusses how conflicts between rights and permissions can lead to UAC prompts during certain administrative operations. The article also explains how UAC works, including the use of standard and administrator access tokens, and when UAC comes into play. Finally, it provide a PowerShell script to check if UAC is enabled.
  • Troubleshoot Runaway Queries in SQL 2014 Using sys.dm_exec_query_profiles The article provides a script for identifying bottlenecks in runaway T-SQL queries in SQL Server. It introduces the sys.dm_exec_query_profiles DMV, which allows viewing of runtime execution plan statistics. This feature can be enabled with “SET STATISTICS PROFILE ON” or “SET STATISTICS XML ON”, providing insights into the query’s execution plan while it’s still running.
  • How to Start or Stop SQL Services without UAC Prompts The article provides a solution for starting or stopping SQL Server & SQL Agent services without User Account Control (UAC) prompts for users with OS administrative rights in an environment where UAC is enabled. It outlines methods to bypass User Account Control (UAC) prompts either by adjusting application settings or using elevated processes.
  • How to Start or Stop SQL Services without OS Admin Rights The article provides a solution for granting non-OS administrators the ability to start or stop SQL Server & SQL Agent services. It suggests using Local Security Templates or Security Configuration and Analysis snap-ins, which can be extended to a Group Policy Object (GPO), allowing users to manage services in a more familiar way without needing special scripts or shortcuts.
  • Duplicate SPN: What is it really? The article explains that a duplicate Service Principal Name (SPN) occurs when the same SPN attribute is found on multiple Active Directory objects, which can cause issues when a client queries Active Directory for a Kerberos connection. It provides a guide on how to identify and remove these duplicates to ensure proper functioning of the system.
  • How to Use an Encrypted Password Within a SQL Agent job to Access a Fileshare in an Untrusted Domain The article provides a solution for using an encrypted password within a Microsoft SQL Server Agent job to access a fileshare in an untrusted domain. The script transforms the clear text password into an encrypted string, storing it in a SQL Server table, and then uses this encrypted password to authenticate to the remote UNC share via a SQL Agent PowerShell job step.
  • PowerShell Lessons Learned from Building an Automated SQL Installation and Patch Management Implementation Provides insights and lessons learned from building an automated SQL installation and patch management implementation using PowerShell. It focuses on the rationale behind using PowerShell for SQL tasks and the experiences from automating SQL deployments.
